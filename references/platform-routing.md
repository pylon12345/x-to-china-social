# 平台与交付模式路由

先定平台，再定交付模式，两者互不混淆。

## 平台

1. CLI 明确指定优先。
2. “只做公众号/不要小红书” → `wechat`。
3. “只做小红书/不要公众号” → `xiaohongshu`。
4. 明确说“双平台/两边都要/公众号和小红书” → `both`。
5. 只出现一个平台名 → 该平台。
6. 未出现或“不同时做” → 默认 `wechat`。

不要因为技能支持双平台就自动生成双份内容。

## 交付模式

- “保存到公众号草稿箱 / 同步公众号 / 保存草稿 / 完整同步” → `full`。
- “快速模式 / 只做本地 / 不要同步 / 本地预览” → `fast`。
- 未说明 → `fast`。
- 小红书任务即使选 `full` 也不生成公众号收据。

`fast` 仍完成内容、配图、公众号本地排版（若选择公众号）和 Obsidian 归档；只跳过远端公众号登录、图片上传与草稿回读。
